# 申诉规则

如果账号主体或第三方认为某条记录存在误收录、证据失效或描述不准确，可以通过 GitHub Issue 提出申诉。

申诉 Issue 需要人工审批，不会触发自动收录流程。

## 申诉时必须提供

1. 平台
2. 平台 UID 或其他稳定唯一标识
3. 账号名称
4. 申诉类型
5. 申诉理由

## 选填信息

- 公开证据链接
- 补充说明

## 处理方式

维护者复核后，可以采取以下处理：

- 保留记录
- 修改账号名称、链接、状态或备注
- 将状态改为 `申诉中`
- 将状态改为 `已移除`
- 关闭申诉 Issue

## 已移除记录

`已移除` 状态的账号不会进入自动生成的三级名单。

## 恶意申诉处理

以下行为可视为申诉流程滥用：

1. 伪造申诉信息。
2. 批量提交无事实基础的申诉。
3. 使用申诉流程骚扰维护者或其他账号主体。
4. 多次提交攻击性、侮辱性或无关内容。
5. 其他明显滥用申诉流程的行为。

经人工确认后，提交者可以被加入 `永久限制/申诉滥用记录.json`。进入该记录后，其后续申诉可以被直接关闭，除非维护者主动决定复核。
