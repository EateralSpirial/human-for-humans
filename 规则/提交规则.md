# 提交规则

普通用户通过 GitHub Issue 提交疑似账号。

## 提交时必须提供

1. 平台
2. 账号名称
3. 平台 UID 或其他稳定唯一标识

如果暂时找不到平台 UID，可以先提交主页链接作为唯一线索。

## 选填信息

以下信息可以选填，维护者后续复核时再补充：

- 主页链接
- 证据链接
- 简短疑似原因

## 有效提交人数计算

1. 同一 GitHub 用户对同一平台账号只计 1 次。
2. 明显灌水、复制、攻击性或无关提交不计入有效提交人数。
3. 维护者可以将无效提交排除在计数之外。
4. 没有证据链接的提交可以先作为线索，但是否计入有效提交人数由维护者复核决定。

## JSON 维护规则

人工审核通过后，维护者把账号写入对应的 `数据源/*.json` 文件。

每条记录至少包含：

```json
{
  "平台UID": "",
  "账号名称": "",
  "排序名": "",
  "主页链接": "",
  "提交人数": 1,
  "证据链接": [],
  "状态": "疑似"
}
```

`排序名` 用于 A-Z 排序。中文账号建议填写拼音，英文账号直接填写小写英文。
