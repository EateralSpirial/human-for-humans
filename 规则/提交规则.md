# 提交规则

普通用户通过 GitHub Issue 提交疑似账号。

## 提交时必须提供

1. 平台
2. 账号名称
3. 平台 UID 或其他稳定唯一标识

如果暂时找不到平台 UID，可以先提交主页链接作为唯一线索。

## 选填信息

以下信息可以选填，维护者后续复核时再补充：

- 主页链接
- 证据链接
- 简短疑似原因

## 自动收录流程

新增 Issue 打开后，GitHub Actions 会自动解析表单内容，并写入对应的 `数据源/*.json` 文件。

自动流程会执行：

1. 读取平台、账号名称、平台 UID。
2. 按平台写入对应 JSON 文件。
3. 同一提交者重复提交同一平台 UID 时，不重复增加提交人数。
4. 自动刷新 `账号名单/一级疑似.md`、`账号名单/二级疑似.md`、`账号名单/三级疑似.md`。
5. 处理完成后自动评论并关闭 Issue。

## 有效提交人数计算

1. 同一 GitHub 用户对同一平台账号只计 1 次。
2. 明显灌水、复制、攻击性或无关提交不计入有效提交人数。
3. 维护者可以将无效提交排除在计数之外。
4. 没有证据链接的提交可以先作为线索，但是否计入有效提交人数由维护者复核决定。

## JSON 维护规则

自动收录或人工审核通过后，账号会写入对应的 `数据源/*.json` 文件。

每条记录至少包含：

```json
{
  "平台UID": "",
  "账号名称": "",
  "排序名": "",
  "主页链接": "",
  "提交人数": 1,
  "证据链接": [],
  "状态": "疑似"
}
```

`排序名` 用于 A-Z 排序。自动收录流程会把中文账号名称转换为拼音排序名；人工编辑时也可以自行填写。
